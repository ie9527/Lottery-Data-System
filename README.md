# 彩票数据系统

基于 **FastAPI** 构建的彩票开奖数据采集、存储、查询与分析平台。支持多彩种历史数据抓取、智能轮询更新、号码统计分析、排行分析、未开奖号码预测及 **AI 智能对话**等功能。

---

## 功能特性

### 🤖 AI 智能对话
- **自然语言查询**：通过 AI 对话直接查询开奖号码、号码统计、历史记录
- **20+ 数据查询工具**：涵盖日期查询、期号查询、号码统计、组选查询、未开号码、复杂条件统计等
- **流式输出**：SSE 实时推送思考过程和回答内容
- **Markdown 渲染**：对话内容支持表格、代码块、列表等 markdown 格式
- **Token 用量统计**：实时显示输入/输出 tokens 及缓存命中情况
- **余额估算**：基于实际用量动态计算费用
- **思考模式**：复杂分析问题自动启用心智链推理

### 数据采集
- **多源数据抓取**：支持 JSON API 和 TXT 文件两种数据源
- **智能轮询更新**：自动检测开奖时间窗口（21:00~23:00），开奖后 8 分钟延迟读取，避免无效请求
- **断服补数据**：启动时自动检测并补齐服务器离线期间遗漏的开奖数据
- **手动更新**：支持手动触发全量更新（1 小时冷却限制）
- **增量更新**：仅拉取数据库中不存在的新期号，避免重复数据

### 数据分析
- **号码统计**：直选 / 组选出现次数统计，覆盖全部彩种
- **排行分析**：单号排行、双号组合排行、三号组选排行、尾号排行、红蓝球排行
- **高级搜索**：支持按期号、日期范围、号码组合等多维度查询
- **未开奖号码**：自动计算指定范围内尚未开出的号码组合
- **历史的今天**：查询历史上同月同日的开奖数据

### 系统管理
- **可视化状态面板**：实时显示轮询状态（绿色脉冲动画）、冷却倒计时、开奖倒计时
- **完整日志系统**：按日轮转的日志文件，记录错误、调试和操作日志，保留 30 天
- **响应式界面**：适配手机端、平板和桌面端

### 1. 克隆仓库

```bash

```

### 2. 安装依赖

```bash
pip install -r requirements.txt
```

### 3. 安装依赖库

```bash
# FastAPI Web 框架
pip install fastapi uvicorn

# 数据库
pip install aiosqlite

# 其他依赖
pip install aiohttp
```

## 启动方式

### 方式一：直接启动（推荐）

首次启动时会自动初始化数据库、构建号码统计并补充断服数据：

```bash
python run.py
```

启动后访问：**http://127.0.0.1:8001**

### 方式二：仅启动服务器（不执行初始化）

```bash
python -m app.main
```

或通过 Uvicorn 直接启动：

```bash
uvicorn app.main:app --host 127.0.0.1 --port 8001 --reload
```

### 启动参数

启动脚本 `run.py` 会自动执行以下流程：

1. 初始化 SQLite 数据库
2. 检查并构建号码统计（首次运行约需 10~20 秒）
3. 执行断服数据补齐检查
4. 启动 Web 服务器

**注意**：所有配置参数集中在 `app/config.py` 中，可按需修改。

## 使用方法

### 访问页面

打开浏览器访问 `http://127.0.0.1:8001`

### 页面导航

| 页面 | 路径 | 功能 |
|------|------|------|
| 首页 | `/` | 开奖概览、倒计时、轮询状态 |
| 开奖记录 | `/{彩种代码}` | 查看历史开奖数据 |
| 号码查询 | `/{彩种代码}/search` | 按号码组合搜索 |
| 高级搜索 | `/{彩种代码}/advanced` | 按期号/日期搜索 |
| 统计次数 | `/{彩种代码}/stats` | 号码出现次数统计 |
| 排行分析 | `/{彩种代码}/ranking` | 号码排行分析 |
| 历史的今天 | `/{彩种代码}/today` | 历史上今日开奖 |
| 未开奖号 | `/{彩种代码}/unused` | 未开出号码查询 |
| 系统日志 | `/system` | 系统状态、轮询日志 |

### API 接口

系统提供完整的 JSON API，所有接口均以 `/api` 为前缀：

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/types` | GET | 获取所有彩种信息 |
| `/api/draws/{code}` | GET | 获取开奖记录 |
| `/api/latest` | GET | 所有彩种最新开奖 |
| `/api/system/status` | GET | 系统状态（冷却+日志+轮询） |
| `/api/countdown` | GET | 开奖倒计时 |
| `/api/update/manual` | POST | 手动触发全量更新 |
| `/api/{code}/stats` | GET | 号码统计概览 |
| `/api/{code}/ranking/query` | GET | 号码排行查询 |

## 核心代码说明

### 配置中心 (`app/config.py`)

所有可配置参数集中管理：服务器端口、数据库路径、开奖时间表、轮询间隔、冷却时间、缓存 TTL 等。修改开奖时间只需编辑 `DRAW_SCHEDULE` 字典。

### 智能轮询 (`app/auto_updater.py`)

- 仅在 **21:00~23:00** 活动窗口内运行
- 逐一检查各彩种开奖条件（开奖日、开奖时间+8分钟延迟）
- 当日全部彩种更新完毕后自动停止
- 跨天自动重置

### 日志系统 (`app/logger.py`)

- 使用 `TimedRotatingFileHandler` 按天轮转日志文件
- 保留 30 天历史日志
- 支持 DEBUG / INFO / WARNING / ERROR / CRITICAL 五级日志
- 同时输出到控制台和文件

### 缓存系统 (`app/cache.py`)

- 轻量级内存缓存，支持 TTL 过期
- 兼容同步和异步函数
- 手动更新时自动失效相关缓存

### 数据采集 (`app/fetcher.py`)

- 支持 JSON API 和 TXT 文件两种数据源
- 自动处理网络异常和超时
- 增量更新，避免重复数据

## 配置说明

主要配置项在 `app/config.py` 中：

```python
SERVER_HOST = "127.0.0.1"    # 服务器地址
SERVER_PORT = 8001           # 服务器端口
POLL_INTERVAL = 600          # 轮询间隔（秒）
POLL_START_HOUR = 21         # 轮询开始时间
POLL_END_HOUR = 23           # 轮询结束时间
POST_DRAW_DELAY = 8          # 开奖后延迟（分钟）
MANUAL_UPDATE_COOLDOWN = 3600  # 手动更新冷却（秒）
```

## 扩展方向

1. **数据可视化**：接入 ECharts / Chart.js 生成号码走势图、冷热分布图
2. **预测模型**：基于历史数据建立号码出现概率预测模型（马尔可夫链、机器学习）
3. **通知推送**：开奖结果推送（邮件、钉钉/企业微信机器人、Server酱）
4. **多用户支持**：添加用户认证系统，支持个性化收藏和订阅
5. **API 文档**：集成 Swagger UI 自动生成交互式 API 文档（FastAPI 原生支持）
6. **定时任务**：使用 APScheduler 替代当前轮询机制，更精确的任务调度
7. **数据导出增强**：支持 Excel/PDF 格式导出，定时自动导出
8. **前端框架升级**：使用 Vue/React 重写前端，提升交互体验
9. **容器化部署**：提供 Docker 镜像，一键部署
10. **多数据源**：接入更多彩票数据 API，提高数据可靠性
